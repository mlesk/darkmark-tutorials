# Git Worktrees — Resources

## Knowledge

- [Official docs: `git-worktree` — Git SCM](https://git-scm.com/docs/git-worktree)
  The authoritative reference. Use for: checking exact flag behavior, understanding edge cases, and as the final word on anything worktree-related.

- [Article: _Git Worktrees: A Practical Guide_ — The GitHub Blog](https://github.blog/open-source/git/git-worktrees-a-practical-guide/)
  GitHub's own guide, grounded in real-world workflows. Use for: a readable, example-driven walkthrough of the common patterns.

- [Article: _Parallel Git Workflows with Git Worktrees_ — Martin Fowler's blog (by Mark Seemann)](https://martinfowler.com/articles/branching-patterns.html#GitWorktree)
  Places worktrees in the context of branching strategy. Use for: understanding where worktrees fit in the broader git workflow landscape.

- [Man page: `git-worktree` — linux.die.net](https://linux.die.net/man/1/git-worktree)
  The Linux man page with examples. Use for: quick command reference with typical invocation patterns.

- [Article: _Working with Git Worktrees_ — Atlassian (Bitbucket blog)](https://bitbucket.org/blog/git-worktrees)
  Concise tutorial from another major git host. Use for: a second take on the basic workflow, with slightly different framing.

- [Video: _Git Merge 2022: Git Worktrees_ — GitButler (YouTube)](https://www.youtube.com/results?search_query=git+worktrees+explained)
  Conference talk level explanation. Use for: a deeper dive when you want to hear someone reason aloud about worktree design.

## Wisdom (Communities)

- [r/git](https://reddit.com/r/git)
  Active subreddit with experienced git users. Use for: edge-case questions, "is this a good idea?" sanity checks, and learning from other people's mistakes.
