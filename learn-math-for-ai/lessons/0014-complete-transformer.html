The Complete Architecture

Input: "The cat sat" → Token IDs: [464, 3797, 6313]

▸ Embedding Layer: token IDs → vectors (d_model = 768) [Lesson 0004]
▸ + Positional Encoding: add position signal [Lesson 0004]

↓ Repeat N=12 times ↓

▸ Multi-Head Attention: tokens exchange information [Lesson 0013]
▸ Add & Layer Norm: residual + stabilize [Lesson 0005]
▸ Feed-Forward Network: per-token transformation
▸ Add & Layer Norm: residual + stabilize [Lesson 0005]

↓ After final layer ↓

▸ Output Projection: final hidden state → logits (vocab_size = 50,257)
▸ Softmax: logits → probability distribution [Lesson 0012]

Output: P("on")=0.73, P("under")=0.11, P(".")=0.03, …

Component-by-Component Walkthrough

1. Tokenization + Embedding (Input → Vectors)

Text is split into tokens (subwords). Each token ID is looked up in the embedding matrix E (vocab_size × d_model). The result is a matrix X (seq_len × d_model). Positional encodings are added (not concatenated) — this is why the dimensions must match. Lessons 0001, 0004.

2. Multi-Head Attention (The Core)

Each token's vector is projected into Q, K, V through learned matrices. Attention computes context-aware representations: each token gathers information from all other tokens, weighted by relevance. Multiple heads run in parallel, capturing different relationship types. Lesson 0013.

3. Add & Norm (Residual + Stabilization)

Residual connection: output = LayerNorm(x + Attention(x)). The "+" preserves the original signal; LayerNorm prevents values from growing/shrinking across layers. Without residuals, deep transformers don't train. Lesson 0005.

4. Feed-Forward Network (Per-Token Processing)

A small two-layer neural network applied independently to each position:

FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂

Typically, the hidden dimension is 4× d_model (e.g., 768 → 3072 → 768). This is where most of the model's parameters live — the FFN layers contain ~⅔ of all parameters. Lessons 0002, 0003.

5. Stack N Layers

The block (Attention → Add&Norm → FFN → Add&Norm) is repeated N times (N=12 for GPT-2 base, N=96 for GPT-3). Each layer refines the representations. Deeper layers capture more abstract patterns. Lesson 0005.

6. Output: Hidden State → Probabilities

The final hidden state at the last position is multiplied by the unembedding matrix (often tied to the embedding matrix: W_unembed = Eᵀ). This produces logits — a vector of length vocab_size. Softmax converts logits to a probability distribution. The model samples from this distribution to generate the next token. Lessons 0007–0012.

Training: How It Learns

During training (not inference), the full forward pass happens, then:

  1. Loss computation: Cross-entropy between predicted distribution Q and true next token P: loss = −log Q(correct_token). Lessons 0010, 0011.
  2. Backpropagation: Gradients flow backward through every operation — softmax, matrix multiplications, embeddings, all of it. The chain rule from calculus (your calculus background!) computes how each parameter contributed to the loss.
  3. Gradient descent: Every parameter (all ~175B of them) is nudged slightly in the direction that reduces loss. The learning rate controls how big the nudge is.
  4. Repeat billions of times across the entire training corpus.
Training is just: forward pass → compute loss (cross-entropy) → backward pass (chain rule) → update parameters (gradient descent) → repeat. All the mathematical machinery you've learned — vectors, matrices, softmax, cross-entropy — is designed to make this loop work efficiently and at scale. The transformer architecture isn't just clever; it's optimized for gradient-based learning.

Beyond Transformers: What Comes Next

The transformer is the foundation, not the end. Here's how your mathematical literacy extends forward:

  • GPT series (decoder-only): drop the encoder, keep the decoder stack. Your math applies directly.
  • BERT (encoder-only): drop the decoder, keep the encoder. Bidirectional attention (no causal mask). Same math.
  • Vision Transformers (ViT): split images into patches, treat each patch as a "token." Same math, different input modality.
  • Diffusion models (DALL·E, Stable Diffusion): add Gaussian noise, learn to reverse it. Your probability foundations (Gaussians, distributions) are the language.
  • State-space models (Mamba, S4): linear recurrence instead of quadratic attention. Your matrix multiplication and linear algebra foundations are the language for comparing these to transformers.
  • Multimodal models (GPT-4V, Gemini): images + text → shared embedding space → transformer. Your embedding space understanding lets you see how different modalities share a representation.

Check Your Understanding

Q1: The feed-forward network in a transformer block is applied independently to each position. Why is this okay, given that attention already mixed information across positions?

A. It's a bug that was never fixed B. Attention handles cross-position mixing; the FFN handles per-position transformation. The division of labor is clean: attention = "communicate", FFN = "think" C. The FFN actually mixes positions internally D. Position-independent processing doesn't work — this is why transformers need many layers

Q2: The embedding matrix and the output projection matrix are often "tied" (W_output = Eᵀ). Why does this make mathematical sense?

A. It saves memory — no deeper reason B. The embedding maps tokens → vectors; the output maps vectors → token probabilities. If a token's embedding is a good representation of its meaning, then using the same mapping in reverse to score how well a vector matches each token is mathematically natural C. The matrices are forced to be tied by the transformer architecture D. Tying matrices always improves performance

Q3 (Capstone): A new paper claims to improve transformers by replacing softmax in attention with a different normalization. What do you now know to ask about this claim?

A. "What programming language did you use?" B. "Does your alternative still produce row-stochastic matrices (non-negative, rows sum to 1)? Does it preserve the order of attention scores? Is it differentiable? Does it avoid saturation like √dₖ scaling does? And does it maintain or improve the convex combination property that stabilizes training?" C. "Is it faster than softmax?" D. "Does it use fewer lines of code?"
