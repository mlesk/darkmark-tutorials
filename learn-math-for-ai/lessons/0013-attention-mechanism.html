0013 — The Attention Mechanism: Step by Step

Convergence · ~30 minutes · The core innovation of the transformer

🧩 Puzzle Piece: This is it. The attention mechanism is where everything you've learned converges. Vectors (0001) become queries, keys, and values through matrix transformations (0002). Matrix multiplication (0003) computes all pairwise similarities. Softmax (0012) turns those similarities into probability distributions. The result is a weighted combination of value vectors — each token selectively gathering information from every other token. Every lesson so far has been building toward understanding this mechanism.
"The cat sat on the mat." How does the model know that "sat" is the verb and "cat" is its subject? By attending: "sat" looks back at all previous words, computes how relevant each one is, and gathers information accordingly. "Cat" gets high attention. "The" gets low attention. This is the attention mechanism — and here is exactly how it works.

The Core Formula

Attention(Q, K, V) = softmax(Q·Kᵀ / √dₖ) · V

Every symbol in this formula has a precise mathematical meaning. You now know all of them:

Symbol What It Is Lesson
Q Query matrix — "what am I looking for?" Each row is one token's query vector 0002
K Key matrix — "what do I contain?" Each row is one token's key vector 0002
V Value matrix — "what information do I carry?" Each row is one token's value vector 0002
Q·Kᵀ Matrix multiplication — all pairwise dot products between queries and keys 0003
√dₖ Scaling factor — prevents softmax saturation in high dimensions 0012
softmax(·) Row-wise softmax — converts similarity scores to probability distributions 0012
·V Weighted average of value vectors, using attention probabilities as weights 0001, 0003

Step-by-Step Walkthrough

Step 1: Create Q, K, V from the input

The input is a matrix X (seq_len × d_model) — one row per token. Three learned weight matrices project X into query, key, and value spaces:

Q = X·WQ   K = X·WK   V = X·WV

Each token now has three representations: what it's looking for (query), what it offers (key), and what it carries (value).

Step 2: Compute all pairwise similarities

Scores = Q·Kᵀ

Entry (i,j) = qᵢ·kⱼ — how relevant token j's content is to token i's query. This is an (seq_len × seq_len) matrix of raw similarity scores.

Step 3: Scale and apply softmax

Weights = softmax(Scores / √dₖ)

Each row becomes a probability distribution over all tokens. Row i tells you: "token i should pay X% attention to token 0, Y% to token 1, Z% to token 2…" The weights in each row sum to 1.

Step 4: Weighted combination of values

Output = Weights · V

Each output row is a weighted average of all value vectors, where the weights are the attention probabilities. Token i's output = Σⱼ (attention weight ij) · vⱼ.

Tokens that got high attention contribute more to the output. Tokens that got near-zero attention contribute almost nothing. The output at position i is a context-aware representation of token i — it's been enriched with information from every relevant token in the sequence.

Multi-Head Attention

Instead of one set of Q/K/V projections, the transformer uses multiple heads (typically 8–128):

MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) · WO
where headᵢ = Attention(Q·WQⁱ, K·WKⁱ, V·WVⁱ)

Each head learns to attend to different things: one head might track syntactic relationships (subject-verb), another might track semantic relationships (synonyms), another might track positional patterns. The heads operate in parallel, and their outputs are concatenated and projected back to the model dimension.

Multi-head attention is like having multiple expert readers. One reader looks for grammar, one for meaning, one for pronoun references. They all read the same text but notice different things. Their insights are combined into a richer understanding than any single reader could provide.

Causal Masking

During autoregressive generation, the model shouldn't see future tokens. A causal mask sets all scores where j > i to −∞ before softmax:

MaskedScoresij = { Scoresij if j ≤ i; −∞ if j > i }

After softmax: exp(−∞) = 0, so future tokens get exactly zero attention weight. The model can only attend to the current token and previous tokens.
