0006 — Eigenvalues, SVD & Matrix Secrets

Linear Algebra Pillar · ~25 minutes · Final LA lesson

🧩 Puzzle Piece: The weight matrices inside a transformer are large (e.g., 4096×4096). Can we compress them without losing what they've learned? Can we decompose them to understand what they "really do"? The answers come from eigenvalues and the Singular Value Decomposition (SVD). This lesson covers the ideas behind techniques like LoRA (Low-Rank Adaptation) — the dominant method for efficiently fine-tuning large language models — and why "low-rank" is a concept that matters for every AI practitioner.
A matrix is a transformation of space. But what does that transformation actually do? Rotate? Stretch? Collapse? The secret: every matrix can be decomposed into rotation → scaling → rotation. This is the Singular Value Decomposition, and it's the universal skeleton key for understanding matrices.

Eigenvectors and Eigenvalues

When a matrix A multiplies a vector v, most vectors change direction. But some special vectors — eigenvectors — only get scaled:

A·v = λ·v

The vector v is an eigenvector. Its direction is unchanged by A. The scalar λ (lambda) is the eigenvalue — how much v is stretched (λ > 1), shrunk (0 < λ < 1), flipped (λ < 0), or annihilated (λ = 0).

Eigenvectors reveal the intrinsic axes of a transformation — the directions that the matrix treats as special.

If a matrix has a zero eigenvalue, it's collapsing some dimension of space. Information is being lost. This matters for AI: when weight matrices have near-zero eigenvalues, they're effectively lower-rank than their dimensions suggest. This is why compression techniques work.

Singular Value Decomposition (SVD)

Not every matrix has a full set of eigenvectors. But every matrix has an SVD:

A = U · Σ · Vᵀ

  • U: orthogonal matrix (rotation/reflection in output space)
  • Σ: diagonal matrix of singular values (σ₁ ≥ σ₂ ≥ … ≥ σᵣ > 0) — how much each dimension is stretched
  • Vᵀ: orthogonal matrix (rotation/reflection in input space)

In words: every linear transformation is: rotate → scale each axis independently → rotate again. The singular values tell you how much each axis gets stretched. If σₖ is tiny, the k-th dimension barely matters — the matrix is effectively lower-rank.

The SVD is the universal decomposition. It works for every matrix — square or rectangular, invertible or not. The singular values tell you the matrix's "effective dimension." If only 32 out of 4096 singular values are large, the matrix is effectively rank-32 — and you can compress it dramatically.

Low-Rank Approximation and LoRA

Keep only the top-k singular values (and corresponding columns of U and V). You get the best possible rank-k approximation of the original matrix, measured by how much information is preserved.

LoRA (Low-Rank Adaptation) exploits this for fine-tuning: instead of updating a full weight matrix W (d×d, millions of parameters), LoRA learns two small matrices A (d×r) and B (r×d) where r ≪ d, and the update is:

Wupdated = Wfrozen + A·B

Since r might be 8 or 16 while d is 4096, the number of trainable parameters drops from d² to 2dr — a reduction of 99.6%+. This is how individuals can fine-tune models like Llama on a single GPU. The mathematical bet: the changes needed during fine-tuning are low-rank — they live in a small subspace of the full weight matrix.

LoRA works because fine-tuning adjustments are low-rank. The pre-trained model already knows a lot. Adapting it to a new task only requires small, structured changes — changes that can be represented as the product of two skinny matrices. This is a direct application of the low-rank approximation principle from SVD.
