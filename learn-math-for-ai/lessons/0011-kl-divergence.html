0011 — KL Divergence: Distance Between Distributions

Probability Pillar · ~20 minutes · Final probability lesson

🧩 Puzzle Piece: Training a language model means making its output distribution Q match the true data distribution P as closely as possible. KL divergence measures exactly this gap: how much information is lost when you use Q instead of P. Minimizing cross-entropy is equivalent to minimizing KL divergence — they differ only by the entropy of the data, which is constant. This is why "training = minimizing KL divergence."
Two probability distributions over the same set of outcomes — how different are they? Not just "are they the same peak?" but "how much information would I lose if I used one instead of the other?" KL divergence answers this — and it's the theoretical foundation of AI training.

Definition

KL(P ‖ Q) = Σ P(x) · log(P(x) / Q(x)) = EP[log(P/Q)]

KL divergence measures the expected log ratio of P to Q, where the expectation is taken under P (the true distribution). Intuitively: for each outcome x, compute how much more probable P thinks x is than Q thinks x is, in log space, then average over P.

Key Properties

  1. KL(P ‖ Q) ≥ 0 — always non-negative. Zero if and only if P = Q everywhere.
  2. KL(P ‖ Q) ≠ KL(Q ‖ P) — it's asymmetric! This is critical. Using Q to approximate P is not the same as using P to approximate Q.
  3. Not a true distance metric — because of the asymmetry and because it doesn't satisfy the triangle inequality. It's a "divergence."
The asymmetry matters for AI. KL(P_data ‖ Q_model) penalizes the model for assigning low probability where the data has high probability (mode-covering). KL(Q_model ‖ P_data) penalizes the model for assigning high probability where the data has low probability (mode-seeking). Different training objectives optimize different KL directions, leading to different model behaviors (e.g., beam search vs. sampling).

Cross-Entropy = Entropy + KL Divergence

H(P, Q) = H(P) + KL(P ‖ Q)

Cross-entropy decomposes into: the unavoidable uncertainty of the data itself (H(P)) plus the extra uncertainty from using the wrong model (KL divergence).

During training, H(P) — the entropy of real language — is constant (it's a property of the data, not the model). So minimizing cross-entropy = minimizing KL divergence. The model can't make language less inherently uncertain; it can only reduce the gap between its predictions and reality.

This completes the training story. The model's loss function is cross-entropy = H(data) + KL(data ‖ model). Since H(data) is fixed, minimizing the loss means minimizing KL divergence — making the model's distribution as close as possible to the true distribution of language. That's what "learning" means mathematically: reducing KL divergence.

Check Your Understanding

Q1: KL(P ‖ Q) = 0. What does this imply?

A. P and Q are completely different distributions B. P = Q for all outcomes — the distributions are identical C. Q is a uniform distribution D. P has zero entropy

Q2: During training, cross-entropy loss decreases from 4.2 to 2.1. What's happening mathematically?

A. The entropy of language H(P) is decreasing B. KL(P ‖ Q_model) is decreasing — the model's distribution is getting closer to the true data distribution C. The model is memorizing the training data exactly D. The vocabulary size is shrinking

Q3: A model assigns P("the") = 0.9 when the true probability in language is 0.3. Another model assigns P("the") = 0.3. Which contributes more to KL divergence, and why?

A. The 0.9 model — because 0.9 > 0.3 B. The 0.9 model — because KL uses the log ratio: log(0.3/0.9) ≈ −1.1, weighted by P=0.3 C. They contribute equally D. Neither — KL divergence only penalizes underestimation, not overestimation
